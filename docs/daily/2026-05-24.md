## Retain
- -> bank/experience: 静态构建产物仓库中若原场景地表由 bundle 程序化生成，替换单个纹理文件不会生效；应先定位真实渲染组件，例如本轮真实入口是 `Ye()` 与 `Je()`，不是背景层 `ie()`。
- -> bank/world: 当前“我的月球”主入口继续沿用原 3D 前端，AI 系统只作为覆盖层；地表纹理已由 `SceneRoot-DLjunhxr.js` 的 `Ye()` / `Je()` 直接引用 `assets/textures/plaza-ground.png`。
- -> bank/experience: 原场景地表的浅色五边形来自 `Xe()` 中 `circleGeometry args:[.9+t%4*.13,5]` 的程序化铺片层；月面替换时应禁用该层而不是改贴图。
- -> bank/experience: 玩家主体黄色小车来自 `Ae()`，真正模型本体来自 `Te()` 加载的 `cape-cat.glb`；移除小车应删玩家组内 `(0,N.jsx)(Ae,{})` 挂载点，不必删函数体。
- -> bank/experience: `circus-tent.glb` 的主体缩放在 `Be()` 的 `Y({... scale:7.4 })`；用户控制模型缩放在 `ke()` 内 `Te({... scale:1.75 })`；顶部红旗不是 GLB 自带，而是 `Be()` 额外插入的 cone/cylinder。
- -> bank/experience: 月面场景里的绿色小草来自 `xe()` 挂载的程序化草场；移除草最稳的是删 `lt()` 里的 `(0,N.jsx)(xe,{playerPositionRef:l})`，而不是逐个改 `ye/be/Se`。
- -> bank/experience: 地表小型彩色三角锥来自 `Qe()` 内部的 `et()`，不是模型资产；移除时直接令 `et()` 返回空组，比删除整层 `Qe()` 更稳，不会误伤树和其他装饰。
- -> bank/experience: 当前首页重资源预热主要来自两处：`SceneRoot-DLjunhxr.js` 的 `O.preload(...)` 会提前拉取多个 GLB，`index-BezMjCv7.js` 的作品 `<video>` 默认 `preload: metadata` 会触发视频元数据请求；要减轻首屏压力，应同时移除前者并把后者改为 `none`。
- -> bank/experience: 外围树阵来自 `ie()` 内的 `e.map(le)`，外围护栏来自 `Ze()`；若只想清掉外圈噪音，应直接断开这两个挂载点，而不是删除中间 `Qe()` 的装饰树。
- -> bank/experience: 中间装饰树来自 `$e()`；要替换成真实模型，可直接让 `$e()` 调用 `Y({ path: A('/assets/models/props/tree.glb') })`。但 `tree.glb` 约 28.7MB，会明显抬高首屏模型负担。
- -> bank/experience: 若月面贴图看起来“没有铺满地面”，优先检查 `Ye()` 的贴图半径是否小于底层圆盘；本轮裸色露出就是 `22.2` 的底盘大于 `20.6` 的贴图圈。
- -> bank/experience: 界面立牌层来自 `at()`，雕像层来自 `ue()`；如果只是要清空界面里的作品牌和雕像，最稳的是直接从 `lt()` 删除这两个挂载点，而不是深入改各自子组件。
