## 我理解的设计

### 1. 单个舱体的形态

* **脊骨形/三角形** ：每个独立舱体像一块脊骨，整体呈三角形态（不是圆柱、不是胶囊、不是方块）
* **三面可连接** ：每个舱体有三个面（对应三角形的三个方向），每个面上都有连接机构
* **连接机构** ：每个面的**下方**有一个 **可伸缩的圆球（凸）** ，需要连接时伸出，不需要时缩回隐藏

### 2. 连接逻辑（榫卯结构）

* **凹 ← 凸** ：一个舱体的凹槽 ← 另一个舱体的圆球凸
* **双向兼容** ：每个面既可以"被凹连接"（作为被动端），也可以"伸出凸连接"（作为主动端）
* **灵活组合** ：
* 单线串联：A-B-C-D... 像一条龙
* 三角组网：三个舱互相连接，形成稳定节点
* 大面积拼接：多个三角节点扩展成网

### 3. 连接后的效果

* **下方** ：圆球凸扣入凹槽，机械锁止
* **上方** ：舱体上边缘紧密贴合，形成连续平面
* **门** ：连接处的上方贴合区域可以开启，变成舱际通道门
* **目标** ：通过拼接，把单个舱的小面积扩展成大面积社区

## 确认后的设计整理

### 1. 舱体形态：立体脊骨三角

**plain**Copy

```plain
        ▲ 顶部（立体脊骨感，有起伏的曲面）
       / \
      /   \    ← 两侧斜面：立体造型，像脊骨的侧翼
     /     \
    /_______\  ← 中间平面：人站立活动区域，平顶
    |       |
    |  门   |  ← 三角形的"凹进去的大面" = 舱体主体的一部分
    |       |      不是平面挖坑，而是整个面内凹形成门洞
    |_______|
      | | |
      底盘（三个支撑点/三个轮子）
```

* **顶部** ：立体脊骨曲面，有设计感，不是平的
* **中部** ：内凹的三角形大面 = 门的位置，人站立高度区域是平的
* **底部** ：底盘，三个支撑点带轮子

---

### 2. 三个面：完全对称通用

每个舱体的三个面 **完全相同** ：

* 每个面都是**内凹的三角形门洞**
* 每个面下方都有 **可伸缩圆球** （平时缩回，需要时从底盘伸出）
* 每个面都可以**凹接凸**或**凸接凹**

---

### 3. 连接机构：下方榫卯 + 上方贴合

 **下方（机械连接 + 能源传输）** ：

**plain**Copy

```plain
[舱体A]          [舱体B]
  │                │
  ▼                ▼
┌─────┐          ┌─────┐
│  门  │          │  门  │  ← 内凹三角形面
│     │          │     │
│  凹  │◄────────►│  凸  │  ← 圆球从底盘伸出，扣入对面凹槽
│     │   圆球    │     │
└─────┘          └─────┘
  ▲                ▲
  │                │
底盘伸出的圆球 ────┘
```

* 圆球从**底盘侧面**伸出，不是从舱体中部
* 平时缩回底盘内部，完全看不到
* 需要连接时，底盘侧面的圆球伸出，扣入对面舱体的凹槽
* 圆球内部有 **触点** ，扣合后传输能源
* **手动确认**后才能并网，不是自动的

 **上方（舱体贴合 + 门）** ：

* 两个舱体的上边缘紧密贴合
* 每个舱体的门都是 **内滑式** （像地铁屏蔽门）
* 连接后，两扇门可以分别开启
* 需要打通时， **同时开两扇门** ，形成通道

---

### 4. 组合方式

**Table**

| 模式               | 示意             | 用途               |
| :----------------- | :--------------- | :----------------- |
| **单线串联** | A-B-C-D          | 移动探索、临时营地 |
| **三角节点** | A-B-C互相连接    | 稳定基站、社区核心 |
| **大面积网** | 多个三角节点扩展 | 社区、城市         |

---

### 5. 底盘与比例

* **三个支撑点** ，每个支撑点带一个轮子（或履带单元）
* 轮子高度 ≈ 舱体站立区域高度，**1:1比例**
* 整体 **扁平但不趴地** ，有科技感和设计感
* 顶部立体脊骨曲面增加视觉高度，实际站立区域适中
